Файл BackUpper.bat отвечает за выгрузку главной ИБ с сервера в текущую директорию
и за последующую ее загрузку в серверную копию.

Порядок работы: программа удаляет предыдущую копию базы (.dt файл) из текущей директории, выгружает ИБ с сервера
в текущую директорию и загружает получившийся .dt файл в другую базу (в копию) на сервере; после этого происходит 
сохранение dt на яндекс диск с очищением предыдущего файла.

ВАЖНО!!! Для правильного сохранения на Яндекс Диске недостаточно удалить предыдущий файл с резервной копией.
Когда файл удаляется так, он отправляется в корзину на облачном хранилище, а не удаляется по-настоящему, в результате чего
место на хранилище не освобождается. Поэтому необходимо очищать корзину ЯД после удаления файла. Это делается скриптом, разрешение
для работы которого дается всего на один год, после чего нужно заново дать ему разрешение на работу в Яндекс Диске (переполучить токен
авторизации - подробнавя инструкция в html файле "инструкция по очистке яндекс диска"). 

ВАЖНО!!! Удалять какие-либо исполняемые файлы (в т.ч. обработку Close1C) нельзя, иначе процесс нарушится. То, что лежит в папке "дополнительно"
можно удалить, но не рекомендуется.

На 23.12.2019 скрипт работает правильно.

Если изменятся какие-либо параметры Базы источника или базы приемника, необходимо
переписать настройки в Bat-е (раздел Настройки выгрузки и загрузки)

IBName - база для выгрузки (Серверный оригинал по задумке)
ServerName - имя сервера с базой источником и приемником (обе базы должны лежать на одном сервере)
UserName - пользователь с правом конфигурирования
Password - пароль пользователя с правом конфигурирования
Platform - путь к исполняемому файлу платформы (рекомендуется использовать Переменную среды)
CopyName - название базы приемника на сервере
